歐盟醫療器材可追溯性全解讀：歐盟 MDR UDI 系統、海關執法實務與 EUDAMED 運作之深度分析
1. 緒論：從 MDD 到 MDR 的數位監管躍進
歐盟醫療器材監管體系隨著《醫療器材法規》（MDR 2017/745）與《體外診斷醫療器材法規》（IVDR 2017/746）的實施，經歷了史無前例的結構性轉型。不同於美國 FDA 長期以來的集中式管理，歐盟過去的指令（Directives）較為分散。然而，新法規（Regulations）引入了強制性的 唯一醫療器材識別系統（UDI） 與集中化的 EUDAMED 資料庫，旨在建立一個全歐盟統一的數位監管基礎設施。
這套系統的核心目標，是確保醫療器材在歐盟單一市場（Single Market）內的供應鏈透明度。對於進口商而言，這意味著海關通關不再僅是稅務問題，而是涉及產品合規性、經濟營運者（Economic Operators）註冊以及嚴格的「上市前驗證」責任的法律程序。
2. 歐盟 UDI 系統的核心架構與 EUDAMED 生態系
歐盟的 UDI 系統雖然遵循全球標準（如 GS1），但在架構上引入了獨有的層級設計，使其比美國系統更為複雜且結構化。
2.1 獨有的識別層級：Basic UDI-DI 與 UDI-DI
歐盟系統將識別碼分為三個關鍵層級，其中 Basic UDI-DI 是歐盟獨創的概念：
基礎 UDI-DI（Basic UDI-DI, BUDI）：
性質：行政與監管的主要索引鍵（Primary Key）。它不會出現在產品實體標籤上。
功能：用於將具有相同預期用途、風險等級、設計及製造特性的「器材家族」群組化。它是歐盟監管的核心，連接了技術文件、歐盟符合性聲明（DoC）以及 EUDAMED 中的證書。
重要性：這是通關文件（符合性聲明）中必須列載的關鍵號碼。
UDI-DI（器材識別碼）：
性質：靜態代碼，印在產品標籤上。
功能：識別特定的產品型號與包裝層級（與美國 FDA DI 概念相同）。
UDI-PI（生產識別碼）：
性質：動態代碼，印在產品標籤上。
功能：包含批號、序號、效期等生產數據。
2.2 EUDAMED：全生命週期的數位中樞
EUDAMED（歐盟醫療器材資料庫）不僅是一個產品目錄，它是一個由六大模組構成的動態生態系統。與美國 GUDID 專注於識別不同，EUDAMED 覆蓋了產品的一生：
行為者註冊（Actor Registration）：管理製造商、授權代表（EC REP）及進口商的身分（SRN）。
UDI 與器材註冊：儲存 Basic UDI-DI 與 UDI-DI 數據。
證書與公告機構（Certificates & Notified Bodies）：追蹤 CE 證書的有效性。
臨床調查與性能研究：管理臨床試驗數據。
警戒與上市後監督（Vigilance & PMS）：處理嚴重事故與現場安全矯正行動（FSCA）。
市場監管（Market Surveillance）：供歐盟成員國主管機關協調執法。
3. 歐盟海關與第 13 條款：進口商的法定驗證責任
與美國由海關（CBP）直接透過 API 掃描攔截的模式略有不同，歐盟的通關執法高度依賴 MDR 第 13 條款（Article 13） 所賦予進口商的「守門人」責任，並配合海關對文件的查驗。
3.1 歐盟符合性聲明（DoC）與通關審查
當醫療器材進入歐盟海關時，海關當局會檢查產品是否具備合法的 CE 標誌 及隨附文件。
關鍵文件：歐盟符合性聲明（EU Declaration of Conformity, DoC）。
UDI 關聯：MDR 要求 DoC 上必須載明該器材所屬的 Basic UDI-DI。海關人員可透過此號碼在 EUDAMED（或尚未完全強制前的過渡系統）中查核該產品是否已註冊、CE 證書是否有效。
3.2 進口商的「第 13 條款」驗證程序
歐盟法規將海關合規的壓力直接轉嫁給了 進口商（Importer）。在產品「投放市場」（Placing on the market）之前，進口商必須進行以下實體與數位驗證（否則即屬違法）：
EUDAMED 查核：確認製造商已在 EUDAMED 註冊並取得 SRN，且該器材（UDI-DI）已登錄。
標籤查驗：確認實體產品已貼上 UDI 標籤，且符合 MDR 附件 VI 的要求。
文件一致性：確認 DoC 上的 Basic UDI-DI 與產品描述相符。
授權代表確認：確認非歐盟製造商已指定歐盟授權代表（EC REP）。
若進口商發現產品不符合上述要求，不得將產品投放市場，並有義務通報製造商及主管機關。
4. 進口商教戰守則：如何準備 EUDAMED 與通關相關資訊
為了確保貨物順利進入歐盟流通，進口商必須建立一套標準作業程序（SOP），以應對 EUDAMED 與海關的雙重檢核。
4.1 單一註冊號碼（SRN）的取得與管理
在歐盟體系下，身分識別的關鍵不是鄧白氏編碼（DUNS），而是 單一註冊號碼（Single Registration Number, SRN）。
進口商自身：必須向所在國的主管機關（Competent Authority）申請成為註冊進口商，取得 SRN。
驗證供應商：進口商必須在 EUDAMED 中驗證製造商與授權代表的 SRN 是否有效。這是海關清關文件中的核心識別資訊。
4.2 產品資料與標籤的預檢
不同於美國僅需關注 DI，歐盟進口商需關注 Basic UDI-DI 的連結性：
DoC 檢查：確保隨貨的符合性聲明中，Basic UDI-DI 清晰無誤。
UDI 載體（UDI Carrier）：檢查包裝上的 UDI 條碼（GS1 或 HIBCC）。MDR 對於不同包裝層級（Unit of Use, Primary, Secondary）有嚴格規定。
標籤資訊：除了 UDI，標籤上必須標示 進口商的名稱、註冊商號或商標，及其聯絡地址。這是海關查驗的重點——進口商必須「實名制」出現在產品或包裝上。
4.3 電子文件與通關準備清單
文件類型	關鍵內容要求	目的與重要性
歐盟符合性聲明 (DoC)	必須包含 Basic UDI-DI、產品名稱、分類、製造商及授權代表資訊。	海關確認產品合法性的核心文件。Basic UDI-DI 是連接 EUDAMED 的鑰匙。
CE 證書副本	由公告機構（Notified Body）簽發的有效證書（針對 Class Is/Im/Ir, IIa, IIb, III）。	證明產品已通過第三方驗證。海關會核對證書編號與 DoC 是否一致。
進口商標示證明	證明進口商資訊已加貼於產品、包裝或隨附文件上的照片或樣張。	滿足 MDR 第 13 條款要求，證明進口商已承擔監管責任。
EUDAMED 註冊證明	相關 SRN 號碼及 UDI-DI 註冊狀態的截圖或匯出檔。	證明供應鏈中的所有行為者（Actors）及器材均已合法登錄。
5. 上市後監督 (PMS) 與警戒系統 (Vigilance)
歐盟的 UDI 系統在產品通關後的上市後監督階段扮演著比美國更為嚴苛的角色。
5.1 趨勢報告與嚴重事故通報
製造商必須利用 Basic UDI-DI 和 UDI-DI 在 EUDAMED 的警戒模組中通報嚴重事故（Serious Incidents）。
進口商角色：進口商通常是當地的第一聯絡點。若收到客訴或事故報告，進口商需協助製造商進行通報。
MIR 表格：歐盟統一的製造商事故報告（MIR）表格強制要求填寫 UDI 資訊。這使得歐盟各國主管機關能透過 EUDAMED 快速共享安全資訊。
5.2 現場安全矯正行動 (FSCA)
當發生召回（Recall）或修正時，製造商需發布「現場安全通知」（FSN）。在歐盟，FSN 必須清楚列出受影響的 UDI-DI 與 UDI-PI。EUDAMED 會將這些資訊向公眾公開（部分模組），讓醫院採購端與病患能透過 UDI 查詢自己的器材是否在召回名單中。
6. 總結：進口商作為合規樞紐
如果說美國 FDA 的 UDI 系統是海關自動化攔截的「數位濾網」，那麼歐盟 MDR 的 UDI 系統則是一張依靠法律責任編織的「契約網」。
在歐盟模式下，海關執法與 MDR 第 13 條款 緊密結合。海關依賴進口商執行實質的合規檢查。對於希望進入歐盟市場的企業而言，取得 SRN、確保 Basic UDI-DI 在文件中的正確性，以及履行進口商的標示與查驗義務，是貨物通關與合法銷售的三大基石。UDI 在此不僅是條碼，更是連接實體貨物與 EUDAMED 數位分身的唯一紐帶，任何環節的數據不匹配（Data Mismatch）都可能導致貨物在邊境受阻或在市場上面臨強制下架的風險。
